安装与基础配置

从零开始安装 Claude Code，了解目录结构，配置 CLAUDE.md，掌握权限模式与启动参数。

什么是 Claude Code

Claude Code 是 Anthropic 官方出品的 AI 编程 CLI 工具，让 Claude 直接在你的终端中运行，能够读写文件、执行命令、调用 Git、调用外部 API——是真正在你本地环境中动手写代码的 AI 助手。

终端原生
直接在 bash/zsh 中运行，无需 GUI，完全 CLI 工作流
文件系统访问
可读写本地文件，理解整个项目上下文
工具调用
内置 Read、Write、Edit、Bash、Grep 等工具
可扩展
通过 MCP、Hooks 无限扩展能力

安装方式

方式一：npm（推荐）

bash
# 全局安装
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version

# 首次运行（会引导认证）
claude

方式二：Homebrew（macOS）

bash
brew install claude-code
💡 认证

首次运行会打开浏览器进行 Anthropic 账号认证。需要有效的 Claude Pro 或 API 账号。

CLAUDE.md 配置详解

CLAUDE.md 是最重要的配置文件，每次对话都会自动加载。它告诉 Claude 关于你项目的一切。

markdown (CLAUDE.md)
# 项目说明
这是一个 React + TypeScript 电商项目。

## 技术栈
- Frontend: React 18, TypeScript, Tailwind CSS
- Backend: Node.js, Express, PostgreSQL
- 测试: Jest, Playwright

## 编码规范
- 使用 2 空格缩进
- 函数命名用 camelCase
- 组件命名用 PascalCase
- 所有新功能必须有测试

## 常用命令
- 启动开发服务器: `npm run dev`
- 运行测试: `npm test`
- 构建: `npm run build`

## 注意事项
- 不要直接修改 dist/ 目录
- 提交前运行 `npm run lint`
- 数据库迁移文件不要手动修改

权限模式

模式说明适用场景
默认（手动）每个敏感操作需用户确认日常使用，安全第一
--dangerously-skip-permissions跳过所有权限检查，完全自动自动化脚本、CI 环境
⚠️ 警告

在生产环境中谨慎使用 --dangerously-skip-permissions，该模式下 Claude 可以无限制执行任何操作。

## Git 规范

Hooks 系统

Hooks 是 Claude Code 的事件驱动自动化系统，让你在特定操作发生时自动执行自定义脚本。

Hooks 原理

Hooks 在特定事件（如工具调用前后、对话开始等）触发，执行用户定义的 shell 脚本。脚本可以：修改行为、记录日志、阻止危险操作、触发通知。

PreToolUse
工具调用前触发。可以阻止操作，修改参数
PostToolUse
工具调用后触发。可以记录结果，触发后续操作
Notification
Claude 发送通知时触发。用于系统通知、消息推送
Stop
Claude 完成响应时触发。用于总结、日志、清理

配置 Hooks（settings.json）

json (.claude/settings.json)
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo '执行 Bash 命令前检查...' && bash ~/scripts/pre-bash-check.sh"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/scripts/auto-format.sh"
          }
        ]
      }
    ],
    "Notification": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "notify-send 'Claude Code' '$CLAUDE_NOTIFICATION_TITLE'"
          }
        ]
      }
    ]
  }
}

Hook 环境变量

变量名说明可用 Hook
CLAUDE_TOOL_NAME当前工具名称（如 Bash, Write）PreToolUse, PostToolUse
CLAUDE_TOOL_INPUT工具输入参数（JSON）PreToolUse
CLAUDE_TOOL_OUTPUT工具执行结果PostToolUse
CLAUDE_NOTIFICATION_TITLE通知标题Notification
CLAUDE_NOTIFICATION_BODY通知内容Notification
CLAUDE_SESSION_ID当前会话 ID所有

实战示例一：自动格式化

每次写入 Python 文件后自动运行 Black 格式化：

bash (~/scripts/auto-format.sh)
#!/bin/bash
# 从环境变量获取工具输入
TOOL_INPUT="$CLAUDE_TOOL_INPUT"

MCP 协议

Model Context Protocol (MCP) 是 Anthropic 定义的开放标准，让 Claude 连接外部工具和数据源。

MCP 是什么

MCP（Model Context Protocol）是一个开放协议，定义了 AI 模型与外部工具之间的通信方式。通过 MCP，Claude 可以：

  • 连接数据库（Postgres、Supabase、SQLite）
  • 控制浏览器（Playwright）
  • 调用外部 API（GitHub、Slack、Jira）
  • 访问文件系统（跨机器、远程）
  • 执行自定义工具逻辑

from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp import types
import httpx

app = Server("my-api-server")

@app.list_tools()
async def list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="get_weather",
            description="获取指定城市的天气信息",
            inputSchema={
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名称"}
                },
                "required": ["city"]
            }
        ),
        types.Tool(
            name="search_database",
            description="搜索产品数据库",
            inputSchema={
                "type": "object",
                "properties": {
                    "query": {"type": "string"},
                    "limit": {"type": "integer", "default": 10}
                },
                "required": ["query"]
            }
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[types.TextContent]:
    if name == "get_weather":
        city = arguments["city"]
        async with httpx.AsyncClient() as client:
            resp = await client.get(f"https://api.weather.com/v1/{city}")
            data = resp.json()
        return [types.TextContent(type="text", text=str(data))]

    elif name == "search_database":
        query = arguments["query"]
        # 实际数据库查询逻辑
        results = [{"id": 1, "name": "示例产品", "query": query}]
        return [types.TextContent(type="text", text=str(results))]

    raise ValueError(f"Unknown tool: {name}")

async def main():
    async with stdio_server() as (read_stream, write_stream):
        await app.run(read_stream, write_stream, app.create_initialization_options())

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())

  tools: [{
    name: "calculate",
    description: "执行数学计算",
    inputSchema: {
      type: "object",

        expression: { type: "string", description: "数学表达式" }
      },

    }
  }]
}));

server.setRequestHandler(CallToolRequestSchema, async (request) => {
  if (request.params.name === "calculate") {

    return { content: [{ type: "text", text: `结果: ${result}` }] };
  }
  throw new Error(`Unknown tool: ${request.params.name}`);
});

const transport = new StdioServerTransport();
await server.connect(transport);

Agent SDK & 多 Agent 系统

使用 Anthropic SDK 构建自主 Agent，实现多 Agent 协作模式，打造复杂的 AI 自动化工作流。

Agent 基本概念

Agent 是能够自主完成复杂任务的 AI 系统，它不只是回答问题，而是规划 → 工具调用 → 自我修正 → 完成任务的完整循环。

规划能力
分析任务，制定执行步骤，处理复杂依赖关系
工具调用
主动使用 read/write/bash 等工具操作真实环境
自我修正
根据执行结果调整策略，处理错误和异常
任务完成
达到目标后自动停止，不会无限循环

基础 Agent 循环（Python 实现）

python
import anthropic, subprocess, json
from pathlib import Path

client = anthropic.Anthropic()

TOOLS = [
    {"name": "read_file", "description": "读取文件",
     "input_schema": {"type":"object","properties":{"path":{"type":"string"}},"required":["path"]}},
    {"name": "write_file", "description": "写入文件",
     "input_schema": {"type":"object","properties":{"path":{"type":"string"},"content":{"type":"string"}},"required":["path","content"]}},
    {"name": "run_cmd", "description": "执行命令",
     "input_schema": {"type":"object","properties":{"cmd":{"type":"string"}},"required":["cmd"]}},
    {"name": "done", "description": "任务完成",
     "input_schema": {"type":"object","properties":{"result":{"type":"string"}},"required":["result"]}}
]

def run_tool(name, args):
    if name == "read_file":
        return Path(args["path"]).read_text(encoding="utf-8")
    elif name == "write_file":
        Path(args["path"]).write_text(args["content"], encoding="utf-8")
        return "写入成功"
    elif name == "run_cmd":
        r = subprocess.run(args["cmd"], shell=True, capture_output=True, text=True)
        return r.stdout + r.stderr

def agent(task, max_iters=20):
    msgs = [{"role":"user","content":task}]
    for i in range(max_iters):
        resp = client.messages.create(
            model="claude-opus-4-6", max_tokens=4096,
            system="自主完成任务。用工具操作。完成时调用 done。",
            tools=TOOLS, messages=msgs
        )
        msgs.append({"role":"assistant","content":resp.content})
        if resp.stop_reason == "end_turn":
            return next((b.text for b in resp.content if hasattr(b,"text")), "完成")
        if resp.stop_reason == "tool_use":
            results = []
            for b in resp.content:
                if b.type == "tool_use":
                    if b.name == "done": return b.input["result"]
                    result = run_tool(b.name, b.input)
                    results.append({"type":"tool_result","tool_use_id":b.id,"content":result})
            msgs.append({"role":"user","content":results})
    return "超过最大迭代次数"

# 使用示例
result = agent("分析当前目录 Python 文件，找出所有函数，生成 functions.md")
print(result)

        messages.append({"role":"assistant","content":resp.content})
        
        if resp.stop_reason == "end_turn":
            return next((b.text for b in resp.content if hasattr(b,"text")), "审查完成")
        
        results = []
        for b in resp.content:
            if b.type != "tool_use": continue
            if b.name == "list_files":
                out = subprocess.run(f"ls {b.input['dir']}", shell=True, capture_output=True, text=True).stdout
            elif b.name == "read_file":
                out = Path(b.input["path"]).read_text(encoding="utf-8", errors="ignore")[:3000]
            elif b.name == "run_tests":
                out = subprocess.run("npm test 2>&1 | head -50", shell=True, capture_output=True, text=True).stdout
            else:
                out = "Unknown tool"
            results.append({"type":"tool_result","tool_use_id":b.id,"content":out})
        messages.append({"role":"user","content":results})

# 使用
report = code_review_agent("./my-project")
print(report)

常见问题 FAQ

Q: Claude Code 和直接用 Claude API 有什么区别？

Claude Code 是 CLI 工具，内置了文件操作、代码执行等工具，专为开发者设计。Claude API 是底层接口，需要自己实现工具调用和对话管理。

Q: 如何减少 Token 消耗？

1) 使用 /compact 压缩长对话；2) CLAUDE.md 保持精简；3) 具体指令比模糊请求省 Token；4) 复杂任务分阶段处理；5) 简单任务用 Haiku 模型。

Q: 如何让 Claude 不重复之前犯过的错误？

使用 /memory 命令将重要约定写入记忆文件，或在 CLAUDE.md 中明确列出禁止事项。记忆会在后续会话中自动加载。

Q: MCP 服务器连接失败怎么办？

检查 settings.json 中的命令路径、环境变量是否正确；用 /doctor 命令诊断；查看 ~/.claude/logs/ 中的错误日志。

Q: 如何在 CI/CD 中使用 Claude Code？

使用 --dangerously-skip-permissions 跳过交互式确认，配合 --print 和 --output-format json 获取机器可读输出。确保 ANTHROPIC_API_KEY 在 CI 环境变量中设置。
